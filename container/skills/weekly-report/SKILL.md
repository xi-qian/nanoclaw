# 周报生成 Skill

## 触发条件

用户提到以下关键词时自动触发本 Skill：
- 「周报」「写周报」「生成周报」「本周周报」
- 任何需要生成或提交周报的消息

## 核心逻辑

### 1. 确定所属项目

读取当前 session 的 CLAUDE.md，从「当前项目」→「项目名称」获取当前项目名称，作为 `所属项目`。

**若 CLAUDE.md 中无项目配置**，或需要填报其他项目，向用户询问需要填报哪个项目的周报。

### 2. 确定汇报人

从消息的 `sender` 字段获取飞书用户名，匹配成员表（tblwxYNzHY559nZu）获取对应人员 ID，作为 `汇报人`。

### 3. 生成进度内容

**第一步：查询本周任务**

本周定义：当前日期往前推到最近周一（Asia/Shanghai 时区）。
以当前日期 2026-03-27 为例，本周为 2026-03-23（周一）至 2026-03-29（周日）。

**第二步：在任务表（tblpllI7IsChbEnA）中筛选：**
- `所属项目` = 当前项目名称
- `状态` = "已完成" 且 `完成时间` 在本周范围内（即 >= 本周一 且 <= 本周日）

**第三步：汇总生成进度内容**
- 列出每个已完成任务的：任务名称、负责人、完成时间
- 如有进行中任务，也一并列出进度摘要

### 4. 获取汇报部门

在成员表（tblwxYNzHY559nZu）中根据汇报人 ID 查询 `部门` 字段：

- **已填写部门**：直接使用该部门名称填入 `汇报部门`
- **未填写或无法查到**：向用户询问「您的部门是？」获得回复后，同时更新成员表中的部门字段（方便后续复用）

### 5. 必填字段清单

| 字段 | 是否需要询问 | 来源 |
|------|------------|------|
| 日期 | 否 | 当前日期（2026-03-27） |
| 汇报人 | 否 | 发送人匹配成员表 |
| 所属项目 | 否（若当前项目存在）| 当前项目名称；无则询问用户 |
| 汇报部门 | 否（若有记录）| 成员表部门字段；无则询问用户 |
| 进度内容 | 否 | 本周完成任务汇总 |
| 风险等级 | **需要询问** | 用户告知（单选，如：高/中/低/无） |
| 风险内容 | 条件询问 | 若风险等级为高/中则填写，低/无可不填 |
| 是否需要协调支持 | **需要询问** | 用户告知（单选，如：是/否） |
| 协调支持内容 | 条件询问 | 若需要协调则填写具体内容 |
| 下周计划 | **需要询问** | 用户告知 |

### 6. 询问缺失字段

若 `所属项目`、`汇报部门`、`下周计划`、`风险等级`、`是否需要协调支持` 等字段未提供，先向用户询问，获得回复后再写入。

### 7. 写入周报表

使用工具写入以下字段：
```
feishu_add_bitable_records
- app_token: "KtdlbOwJeav22zsJakzcMvEPnHb"
- table_id: "tbljIOzAo1unLxqT"
- records: [{
    fields: {
      日期: <当前日期时间戳>,
      汇报人: [{"id": "<人员ID>", "name": "<人员名>"}],
      所属项目: "<当前项目名称>",
      汇报部门: "<部门名称>",
      进度内容: "<本周完成任务汇总文本>",
      风险等级: "<高/中/低/无>",
      风险内容: "<风险具体描述（如有）>",
      是否需要协调支持: "<是/否>",
      协调支持内容: "<协调支持具体内容（如有）>",
      下周计划: "<下周计划文本>"
    }
  }]
```

## 执行步骤

1. 确定当前项目名称（从 CLAUDE.md），若无可用项目则询问用户
2. 确定汇报人（从发送人匹配成员表）
3. 查询本周完成任务，生成进度内容
4. 在成员表中查询汇报人部门，若无则询问用户，并同步更新成员表
5. 检查必填字段是否齐全，缺失则询问用户
6. 写入周报记录
7. 向用户确认周报已生成
