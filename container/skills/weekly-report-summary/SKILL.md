# 周报汇总 Skill

## 功能说明

从项目管理表读取最新数据，保存到本地文件，用5天内更新的项目进行过滤，汇报其中的风险和待协调事项。

- 项目表 table_id: tblW0xITjmuIuTZb
- 飞书共享链接: https://shaehv5cfo.feishu.cn/share/base/view/shrcn134e8OBQD2ER2pkeoLtDNg

## 操作流程

### 第一步：实时读取项目管理表

调用 feishu_list_bitable_records，读取项目表所有记录。

**禁止凭记忆或缓存数据操作，每次必须重新读取。**

### 第二步：保存数据到本地

将读取到的记录列表保存为 `/workspace/group/bitable_data.json`。

需要提取的关键字段（平铺为简单结构）：
- id: 记录ID
- 编号: 数字
- 项目名称: 文本
- 更新日期: 数字（毫秒时间戳）
- 项目风险: 单选文本
- 风险内容: 文本
- 待协调事项: 文本

### 第三步：运行过滤脚本

执行 Python 脚本 `filter_projects.py`（当前目录下），使用系统当前时间计算5天前的时间戳作为过滤条件。

过滤条件：`更新日期 >= 五天前时间戳（毫秒）`

### 第四步：汇报过滤结果

根据过滤后的项目列表，汇报每个项目的：
- 项目编号
- 医院名称（从项目名称中提取，格式为"医院 | 项目名"取第一段）
- 更新日期（转换为 YYYY-MM-DD 格式）
- 项目风险
- 风险内容（若非空）
- 待协调事项（若非空）

**仅汇报有风险或有待协调事项的项目；若两者都为空，简要标注"无"。**

## 注意事项

- 日期字段使用毫秒时间戳，换算时使用北京时间（UTC+8）
- 过滤以任务执行当天为基准，往前计算5天
- **核心原则：必须实时读取数据，禁止凭记忆操作**
