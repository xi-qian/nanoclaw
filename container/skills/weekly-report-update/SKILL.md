# weekly-report-update

将用户描述的周报内容更新到飞书周报表。

## 触发关键词

用户提到以下关键词时触发此 Skill：
- 更新周报
- 跟踪周报
- 添加周报
- 提交周报
- 周报记录
- 写周报

## 执行步骤

### 第一步：解析用户内容

从用户描述中提取以下字段：

| 字段 | 说明 | 获取方式 |
|------|------|---------|
| 所属项目 | 项目名称，需与周报表中的项目名一致 | 从用户描述中识别，留空需要和用户确认 |
| 日期 | 汇报日期，毫秒时间戳 | 默认为当天北京时间，可用"YYYY-MM-DD"描述 |
| 汇报人 | 汇报人姓名 | 从描述中识别人名，如没有则默认是发送者姓名 |
| 汇报部门 | 汇报人所属部门 | 根据用户的openid进行查询，如果查询不到需要和用户确认 |
| 汇报标题 | 格式"YYYY-MM-DD 姓名" | 由日期和汇报人拼接 |
| 进度内容 | 本周进展描述 | 从用户描述中提取核心进展 |
| 风险等级 | 无风险 / 低风险 / 中风险 / 高风险 | 从描述中识别风险描述 |
| 风险内容 | 风险具体说明 | 从描述中提取（选填） |
| 协调支持内容 | 需要协调的事项 | 从描述中提取（选填） |
| 下周计划 | 下周工作计划 | 从描述中提取（选填） |
| 是否需要协调支持 | 是 / 否 | 由是否有协调内容决定 |

### 第二步：日期转换为毫秒时间戳

使用 Python 将日期转换为毫秒时间戳（北京时区）：

```python
from datetime import datetime, timezone, timedelta
dt = datetime(年, 月, 日, tzinfo=timezone(timedelta(hours=8)))
ts = int(dt.timestamp() * 1000)
```

### 第三步：添加到周报表

调用 `feishu_add_bitable_records` 添加记录：

```
app_token: KtdlbOwJeav22zsJakzcMvEPnHb
table_id: tbljIOzAo1unLxqT
```

**TextField 格式**（直接填字符串，不要数组格式）：
- 所属项目、汇报人、汇报部门、汇报标题（普通文本）
- 进度内容、风险内容、协调支持内容、下周计划 → 普通字符串

**DatetimeField**：
- 日期 → 毫秒时间戳

**SingleSelect**：
- 风险等级 → "无风险" / "低风险" / "中风险" / "高风险"
- 是否需要协调支持 → "是" / "否"

### 第四步：确认结果

添加成功后，用自然语言向用户确认已记录的字段内容。

## 注意事项

- TextField 格式为普通字符串，不要用 `[{"text":"","type":"text"}]` 格式
- 日期必须为毫秒时间戳（13位整数）
- 风险等级必须从预设选项中选择
- 汇报标题格式固定为 "YYYY-MM-DD 姓名"
- 如果用户描述中包含多条项目内容，拆分为多条记录分别添加
