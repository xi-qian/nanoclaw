# weekly-report-reminder

## 执行步骤

### 第一步：提取员工姓名

从用户消息中识别要查询的员工姓名。例如：
- 「提醒徐亚平填周报」→ 姓名 = 徐亚平
- 「催张立交周报」→ 姓名 = 张立

如果无法确定姓名，询问用户。

### 第二步：读取周报表

调用 `feishu_list_bitable_records` 读取周报表全部记录：

```
app_token: KtdlbOwJeav22zsJakzcMvEPnHb
table_id: tbljIOzAo1unLxqT
```

### 第三步：筛选该员工的记录

在本地用 Python 筛选 `汇报人 == 姓名` 的记录，并按日期倒序排列。

取最近 **7 天内** 的记录作为「上周周报」。

### 第四步：按项目汇总

将记录按 `所属项目` 分组，每个项目提取：
- 项目名称（提取医院名称，去掉过长管线名）
- 进度内容
- 风险等级
- 风险内容（如有）

### 第五步：检查本周是否已填写

查看该员工在 **今天及今天之前** 是否已有本周（最新日期）的周报记录：
- 如果有 → 整理内容后直接呈现，不重复提醒
- 如果没有 → 在消息末尾追加提醒

### 第六步：生成消息

消息格式示例（以徐亚平为例）：

```
徐亚平你好，上周（04-14 ~ 04-18）周报回顾如下：

• 北京积水潭医院 | 3D MSC 滑膜炎
  方案已基本撰写完成，实验设计改为I+II期，增加样本量及主要疗效观察指标

• 中山大学附属第一医院 | ModCell-001 动脉硬化
  研究方案材料已全部交付，企业主体变更已沟通无异议

• 北京市胸科医院 | EV 肺纤维化
  伦理补充材料完善，EDC监测指标已完成

• 北京积水潭医院 | FGF18 骨关节炎
  暂无更新

• 上海瑞金医院 | MSC 呼吸系统
  瞿书记约见，明确下周拜访时间，预备合作方案

风险关注：中山一院（付炜授权，中风险）、北胸科（胎盘知情同意合规性，中风险）

---

本周周报（截至04-20）还未看到你提交，请记得在周五前填写更新。如有进展可以直接发给我，我帮你录入。
```

**消息原则：**
- 用自然语言，不要太生硬
- 项目名称精简（保留医院名 + 简称）
- 进度内容适当精简，保留核心信息
- 风险项目单独列出提醒
- 提醒语气友好，给出截止时间（周五）

### 第七步：发送消息

根据用户指定的发送方式，分两种情况处理：

**A. 私信发送**

如果用户明确说「私信发给 {姓名}」「私聊提醒 {姓名}」等，使用 `feishu_send_to_user` 发送私聊消息：
- 需要获取对方的 open_id，可通过飞书用户搜索或从对话上下文中提取
- msg_type 用 `post`（富文本，支持 Markdown）

**B. 群里发送（默认行为）**

如果用户在群里执行且未指定私信，**必须 @ 目标用户**：
- 使用 `mcp__nanoclaw__send_message` 发送消息到当前 chat_id
- 消息开头必须使用 `<at user_id="{对方 open_id}"></at>` 格式 @ 对方
- 对方 open_id 获取方式：
  1. 从飞书多维表格成员表（app_token: KtdlbOwJeav22zsJakzcMvEPnHb, table_id: tblwxYNzHY559nZu）中通过姓名匹配获取 open_id
  2. 成员表中「飞书用户」字段包含用户的 open_id 信息
  3. 若成员表中查不到，询问用户提供目标用户的 open_id

## 注意事项

- 周报表中 `汇报人` 字段是文本，直接字符串匹配
- 日期字段是毫秒时间戳，需要转换为日期进行判断
- 如果该员工上周没有任何记录，回复「未找到 {姓名} 上周的周报记录」
- 如果项目名过长，只保留医院名称和关键词（如去掉完整管线名称）
- 风险为「无风险」的项目可以简化或不列风险
