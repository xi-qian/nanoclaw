# 项目周报同步 Skill

## 功能说明

从周报表中读取各项目的最新周报记录，
与项目管理表的更新日期对比，仅同步日期更新的项目到项目表。

## 数据源
- 周报表 app_token: KtdlbOwJeav22zsJakzcMvEPnHb
- 周报表 table_id: tbljIOzAo1unLxqT
- 项目表 app_token: KtdlbOwJeav22zsJakzcMvEPnHb
- 项目表 table_id: tblW0xITjmuIuTZb

## 同步字段
- 更新日期（日期字段，毫秒时间戳）
- 项目进展（文本）
- 项目风险（单选：无风险/低风险/中风险/高风险）
- 风险内容（文本）
- 待协调事项（文本）

## 操作流程

### 第一步：读取周报表（必须每次实时读取）
调用 feishu_list_bitable_records 读取周报表，获取所有记录。
**禁止凭记忆或缓存数据操作，每次必须重新读取。**

### 第二步：按项目分组，取最新记录
对每个项目名称，筛选出该项目的所有周报记录，
按日期字段降序排列，取第一条即为最新记录。
**必须逐一核对每条周报的日期字段，找出每个项目的最新日期。**

### 第三步：读取项目表（必须每次实时读取）
调用 feishu_list_bitable_records 读取项目表，
提取每个项目的"项目名称"和"更新日期"字段。
**禁止凭记忆操作，必须实时读取。**

### 第四步：项目名称精确匹配
周报表与项目表的项目名称字段完全一致，直接用项目名称进行精确匹配。
若周报表中"所属项目"字段为"IDO项目"，跳过（不匹配）。
若周报表记录无"所属项目"字段，跳过。

### 第五步：逐一比对日期
对于每个项目，**必须逐一核对**：
1. 从周报表中找到该项目最新周报的日期
2. 从项目表中找到该项目的当前更新日期
3. 仅当周报日期 > 项目更新日期时，才执行同步
4. 如果项目更新日期为空/null，默认需要同步

**禁止凭记忆判断日期，必须逐条核对实时数据。**

### 第六步：构造并更新同步数据
从最新周报记录中提取：
- 汇报时间 → 更新日期
- 进度内容 → 项目进展
- 风险等级 → 项目风险
- 风险内容 → 风险内容（若无则留空）
- 协调支持内容 → 待协调事项（若无则留空）

调用 feishu_update_bitable_record 更新各记录。

### 第七步：缺少周报记录时（可选）
若周报表中某项目**没有对应周报记录**，而用户希望手动添加：
- 调用 `feishu_add_bitable_records` 添加新记录
- 必填字段：所属项目（与项目表项目名称一致）、汇报时间（毫秒时间戳）
- 可选字段：进度内容、风险等级、风险内容、协调支持内容

## 可用的 MCP 工具

| 工具名 | 功能 |
|--------|------|
| `feishu_list_bitable_records` | 查询表格记录 |
| `feishu_update_bitable_record` | 更新指定记录 |
| `feishu_add_bitable_records` | 批量添加记录（最多 500 条） |

## 注意事项

- 飞书多维表格 URL 格式：
  app_token = /base/{URL路径中的值} = KtdlbOwJeav22zsJakzcMvEPnHb
  table_id = ?table={URL中的参数值}
  勿将 table_id 当作 app_token

- 周报表中"所属项目"为"IDO项目"的记录为通用项目，跳过

- 项目表中"项目名称"为"IDO项目"的记录跳过

- 周报中无对应项目的记录，跳过（不填更新日期）

- 文本字段使用纯字符串格式，不要使用 [{"text":"","type":"text"}] 格式

- 日期字段使用毫秒时间戳（如 1775145600000 = 2026-04-03）

- **核心原则：必须实时读取数据，禁止凭记忆操作**
