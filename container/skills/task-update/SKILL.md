# 任务状态更新 Skill

## 触发条件

用户提到以下关键词时自动触发本 Skill：
- 任务状态变更：「开始了」「完成了」「卡住了」「暂停了」「交付了」「未完成」等
- 任何涉及更新任务记录的消息

## 核心逻辑

### 1. 识别发送人

从消息的 `sender` 字段获取飞书用户名（如「钱希」「冯晓」）。

### 2. 识别任务

**第一步：根据当前项目筛选**

1. 读取当前 session 的 CLAUDE.md，从「当前项目」→「项目名称」获取当前项目名称
2. 在任务表中筛选 `所属项目` = 当前项目名称的记录

**第二步：在筛选结果中匹配任务**

在上述筛选结果中搜索消息中提到的任务名称关键词，精确或包含匹配到对应记录。

### 3. 判断目标状态

| 用户表述 | 目标状态 |
|---------|---------|
| 开始了、启动了 | 进行中 |
| 完成了、搞定了、交付了 | 已完成 |
| 卡住了、遇到问题了 | 风险 |
| 暂停了、挂起了 | 已暂停 |
| 未完成 | 未完成 |

### 4. 更新字段规则（重要）

**以当前日期为基准计算时间：**

- **状态变为「进行中」时**：
  - `状态` = "进行中"
  - `开始时间` = 当前日期时间戳（当日 00:00:00）
  - 若用户提到周期（如"预计3天"），则 `截止时间` = `开始时间` + 周期天数

- **状态变为「已完成」时**：
  - `状态` = "已完成"
  - `完成时间` = 当前日期时间戳（日期字段 `fldXWmbIgI`）

- **仅更新截止时间时**：
  - `截止时间` = 当前日期 + 用户所述周期

## 飞书表格操作工具

使用以下 MCP 工具：

**查询任务记录：**
```
feishu_list_bitable_records
- app_token: "KtdlbOwJeav22zsJakzcMvEPnHb"
- table_id: "tblpllI7IsChbEnA"
- page_size: 50
```

**更新任务状态：**
```
feishu_update_bitable_record
- app_token: "KtdlbOwJeav22zsJakzcMvEPnHb"
- table_id: "tblpllI7IsChbEnA"
- record_id: <匹配到的记录ID>
- fields: { 状态: "<目标状态>", 开始时间: <时间戳>, 截止时间: <时间戳>, 完成时间: <时间戳（仅已完成时）> }
```

## 执行步骤

1. 识别发送人和任务名称
2. 查询匹配的任务记录
3. **判断任务是否存在**：
   - **已存在**：直接进入步骤 4
   - **不存在**：先向用户确认是否新建任务（如：「表格中未找到「XXX」任务，是否需要新增？」），获得确认后再新建
4. 判断目标状态和时间
5. 调用工具更新（已有任务）或新增（新建任务）
6. 向用户确认更新结果

## 新增任务规则

当用户在表格中未找到对应任务，需确认后新建时：

**新增任务所需字段：**
- `任务`：任务名称（文本）
- `所属项目`：读取当前 session 的 CLAUDE.md 文件，从「当前项目」-「项目名称」字段获取项目名称，默认填入该项目名称
- `负责人 (人员 )`：根据发送人匹配成员表（tblwxYNzHY559nZu），填入对应人员 ID
- `状态`：根据用户表述判断（见状态映射表）
- `开始时间`：当前日期时间戳
- `截止时间`：开始时间 + 用户所述周期（如未提及则不填）
- `优先级`：如未提及则默认 P2

**新增任务工具：**
```
feishu_add_bitable_records
- app_token: "KtdlbOwJeav22zsJakzcMvEPnHb"
- table_id: "tblpllI7IsChbEnA"
- records: [{ fields: { 任务: "<任务名>", 所属项目: "<当前项目名称（从CLAUDE.md读取）>", 状态: "<状态>", 开始时间: <时间戳>, 截止时间: <时间戳>, "负责人 (人员 )": [{"id": "<人员ID>", "name": "<人员名>"}] } }]
```
